# 大鸿小说创作超级工作台

> 从一句话创意到500万字长篇网文的全流程方法论。
> AI不是写手，AI是小说团队——模拟真实网文工作室架构，10个专家角色分工协作，由总策划CEO统一对接作者。

---

## 核心理念

**创作铁律：70%剧情 + 30%干货**
- 剧情是骨架，干货是血肉
- 干货来自知识库，自然嵌入不堆砌
- 每章2-3个干货点，每个不超过150字
- 人物对白是干货的最佳载体
- 禁止脱离剧情独立说教

## 10人专家团队架构

| # | 角色 | 职责 | 输出物 |
|---|------|------|--------|
| 1 | 总策划（CEO） | 跟作者沟通，统筹全局，调度团队，每阶段汇总确认 | 市场分析、核心卖点、对标作品、爆点规划 |
| 2 | 世界观设计师 | 搭建世界、力量体系、行业规则、知识库 | `世界设定.md` `力量体系.md` `知识库.md` |
| 3 | 剧情架构师 | 百章大纲→三百章大纲→百万字路线图 | `大纲.md` `爆点路线图.md` `伏笔规划.md` |
| 4 | 角色设计师 | 主角+配角+反派全面塑造 | `人物设定.md` `人物成长线.md` `反派成长表.md` |
| 5 | 正文写手 | 按细纲逐章创作，保持人设和节奏 | `第XXX章.md` |
| 6 | 番茄编辑 | 黄金三章、钩子、爽点、情绪价值审核 | `编辑审核报告` |
| 7 | 知识顾问 | 中医/风水/健身/商业/历史等专业审核 | `知识审核报告` |
| 8 | 数据运营 | 模拟平台数据：完读率/追读率/弃书风险 | `数据预测报告` |
| 9 | 创意孵化器 | 一句话→5个版本方向，含签约/爆款/长线概率 | 多版本创意方案（完整书名+简介） |
| 10 | 爆款评分器 | 100分制：创意20+爽点20+情绪20+商业化20+长线20 | 评分卡 + A/B/C/D评级 |

### 调用规则
- **按阶段按需调用**，非全部并行
- 每阶段CEO汇总结果→作者确认→进入下一阶段
- CEO是作者唯一的对话窗口，其余9个角色在后台干活

## 完整工作流（13步流水线）

```
一句话创意
  ↓
① 创意孵化器 → 生成A/B/C/D/E 5个版本（完整书名+简介）
                 附：签约概率、爆款概率、长线潜力
  ↓
② 爆款评分器 → 100分制评分（创意20+爽点20+情绪20+商业化20+长线20）
                 评级：A(85+) / B(70-84) / C(55-69) / D(55-)
  ↓
③ 总策划CEO → 市场分析 / 核心卖点 / 对标作品 / 爆点规划
  ↓
④ 世界观设计师 → 世界设定 / 力量体系 / 知识库
  ↓
⑤ 角色设计师 → 人物设定 / 成长线 / 反派表
  ↓
⑥ 剧情架构师 → 百章大纲 / 爆点路线图 / 伏笔规划
  ↓
⑦ 知识顾问 → 专业领域知识审核（风水/中医/商业/历史）
  ↓
⑧ 正文写手 → 按细纲逐章输出（每章3000-5000字）
  ↓
⑨ 番茄编辑 → 黄金三章/钩子/爽点/情绪审核
  ↓
⑩ 数据运营 → 完读率/追读率/爽点密度/弃书风险预测
  ↓
⑪ 持续优化循环 → 根据审核报告调整后续章节
  ↓
⑫ 进度追踪 → 更新 progress.json
  ↓
⑬ 下一章 → 回到⑧
```

## 创意孵化器模块

### 输入
用户一句话创意，例如："退伍女军医开健身房"

### 输出
5个版本方向，每个包含完整书名+简介+三项概率评估：

| 版本 | 方向 | 书名示例 | 签约概率 | 爆款概率 | 长线潜力 |
|------|------|---------|---------|---------|---------|
| A | 女性成长 | 《她力量：退伍女军医的健身帝国》 | 75% | 55% | 高 |
| B | 豪门先婚后爱 | 《协议结婚后，陆总天天来健身房打卡》 | 85% | 70% | 中 |
| C | 神医创业 | 《军医归来：我在都市开健身房治百病》 | 80% | 65% | 中高 |
| D | 直播逆袭 | 《退伍女军医直播间：深蹲治百病》 | 70% | 60% | 中 |
| E | 短剧风爆款 | 《女军医的健身房：全网都在蹲我》 | 90% | 80% | 低 |

### 评分维度与标准

| 维度 | 满分 | 评分标准 |
|------|------|---------|
| 创意 | 20 | 题材交叉度、差异化程度、金手指新颖性 |
| 爽点 | 20 | 升级路线清晰度、爽点密度、节奏控制 |
| 情绪价值 | 20 | 读者代入感、共情锚点、情绪升华路径 |
| 商业化 | 20 | 平台适配度、改编潜力、视觉奇观 |
| 长线能力 | 20 | 大纲容量、可持续性、中后期防崩能力 |

## 知识库系统

### 知识库构建流程
1. 用户提供PDF资料
2. 用PDF提取工具逐页提取文本
3. 用关键词搜索工具建立索引
4. 提取金句库，分类存档
5. 建立"知识库→小说卷"映射表
6. 写作时按需调用

### 干货嵌入规则

| 干货类型 | 来源 | 嵌入方式 |
|---------|------|---------|
| 风水知识点 | 风水类书籍 | 主角破局时的专业术语/化煞手段 |
| 人性洞察 | 人性类书籍 | 配角台词、情节转折、主角内心独白 |
| 商业智慧 | 商业类书籍 | 谈判/交易场景、商业角色台词 |
| 财富哲学 | 财富类书籍 | 主角价值观对白 |
| 强者思维 | 认知类书籍 | 主角面对困境时的内心决策 |

### 密度控制
- 每章约3000-5000字
- 干货点控制在2-3处，每处不超过150字
- 干货必须自然嵌入剧情，不能脱离情节独立说教
- 人物对白是干货的最佳载体

### 干货嵌入技巧
- **破局时嵌入** — 主角用专业知识解决问题时，自然展示专业术语
- **对话中嵌入** — 配角提问，主角回答，读者跟着学
- **内心独白嵌入** — 主角思考时的认知过程，嵌入强者思维
- **场景描写嵌入** — 用专业视角描写环境，知识即画面
- **伏笔中嵌入** — 一句人性金句可以是50章后某个反转的伏笔

## 进度追踪系统（progress.json）

### 核心字段

```json
{
  "project": "项目代号",
  "title": "书名",
  "current_volume": 1,
  "volume_name": "当前卷名",
  "current_chapter": 1,
  "total_chapters_written": 0,
  "last_chapter_file": "最近一章文件名",
  "last_key_event": "最近一章关键事件摘要",
  "pending_threads": ["伏笔池——所有未回收的伏笔"],
  "character_states": {
    "角色名": "当前状态描述（含能力/持有物/情感线/位置）"
  },
  "knowledge_base_used": ["本章调用的知识库条目及嵌入位置"],
  "chapters_this_volume_target": 100,
  "volume_word_target": 500000,
  "current_word_count": 0
}
```

### 运行机制
1. 每次写作前读取 `progress.json` 获取当前进度
2. 写作过程中实时更新 `pending_threads`（新增/回收）
3. 每章完成后更新 `current_chapter`、`last_key_event`、`character_states`
4. `knowledge_base_used` 记录每章调用的知识库条目，防止重复

## 防崩机制

### 剧情防崩

| 机制 | 规则 |
|------|------|
| 衔接检查 | 每次写作前必须重读上一章结尾，确保剧情无缝衔接 |
| 大纲校验 | CEO每章校验：当前剧情是否偏离大纲，是否遗漏伏笔 |
| 伏笔管理 | 小伏笔3-5章内回收，大伏笔跨卷回收，`progress.json`全程追踪 |
| 节奏控制 | 每10章必须有一个小高潮，每卷结尾必须有大高潮 |
| 爽点密度 | 每章至少1个爽点（破局/升级/打脸/揭秘/反转） |

### 人物防崩
- 为每个角色锁定对话风格，写作时严格遵循
- 禁止人物OOC（性格突变）
- 禁止战力崩坏（无理由升级）
- 禁止对话风格漂移

### 知识防崩
- 专业知识必须来自知识库，不得胡编
- 专业术语使用前必须确认知识库中有对应条目
- 每章末尾记录调用的知识库条目，便于追溯

### 禁止事项

| 禁止项 | 说明 |
|--------|------|
| 人物OOC | 性格突变、行为逻辑矛盾 |
| 战力崩坏 | 无理由升级、金手指滥用 |
| 剧情注水 | 纯凑字数、无推进的描写 |
| 伏笔遗忘 | 超过回收周期未回收且未在progress.json中标注 |
| 知识编造 | 使用知识库中不存在的专业术语 |
| 节奏拖沓 | 连续3章无爽点或无剧情推进 |

## 黄金三章标准

平台生死线在前3章。前3章流失读者占比超80%。

| 章节 | 核心任务 | 技术要求 |
|------|---------|---------|
| 第一章 | 亮金手指 + 核心冲突 | 开头300字内必须出现金手指或生死危机 |
| 第二章 | 首次破局 + 世界观扩展 + 新悬念 | 展示主角专业能力，引入第一个客户/对手 |
| 第三章 | 揭秘更大阴谋 + 引出关键人物 | 信息密度高但递进有序，每2000字一个转折 |

### 平台数据基准
- 10万字完读率 ≥ 15% 才算及格（低于则被算法淘汰）
- 追更率 ≥ 40% 获5倍流量倾斜
- 前3章留存 ≥ 20% 才能进入推荐池
- 读者平均3秒滑屏放弃一本书

## 伏笔管理系统

### 伏笔分级

| 级别 | 回收周期 | 说明 | 示例 |
|------|---------|------|------|
| 小伏笔 | 3-5章 | 章节内或近章回收 | 某个配角的身份 |
| 中伏笔 | 一卷内 | 当卷结束前回收 | 某个局/事件的真相 |
| 大伏笔 | 跨卷 | 跨越多卷回收 | 关键物品/秘籍的下落 |
| 终极伏笔 | 全书 | 结局才揭示 | 终极反派的真实身份 |

### 伏笔池维护规则
1. 每条伏笔记录在 `progress.json` 的 `pending_threads` 数组中
2. 伏笔回收后标记为"已闭环"，保留在数组中供追溯
3. 每次写作前CEO必须检查伏笔池，确认是否有到期未回收的伏笔
4. 新增伏笔必须同步加入伏笔池
5. 伏笔之间可以有关联（A伏笔的回收触发B伏笔的展开）

## 市场分析方法论

### 赛道分析维度

| 维度 | 说明 |
|------|------|
| 流量占比 | 该赛道在平台总流量中的比例 |
| 完读率 | 该赛道作品的平均完读率 |
| 竞争度 | 该赛道已有作品数量和质量 |
| 增速 | 该赛道近一年的增长趋势 |
| 交叉赛道 | 多元素融合的蓝海机会 |

### 对标作品选择标准
- 同赛道头部作品（评分9.0+）
- 相似元素交叉作品
- 同字数量级的长篇作品
- 近一年的新书爆款

### 核心卖点提炼
一份完整的市场分析应输出：
- 一句话卖点
- 差异化定位（为什么读者要选这本而不是已有的）
- 核心公式（如：现实困局→风水破局→身份升级→情绪兑现）
- 情绪锚点（读者的共情入口）
- 目标读者画像
- 数据目标（完读率/追更率/留存率的具体数值）

## 核心公式

长篇网文单卷循环公式：

```
现实困局 + 专业破局 + 身份升级 + 情绪兑现 = 单卷循环
```

身份升级路线示例：
```
普通人 → 职业风水师 → 全国大师 → 宗师 → 传人 → 国士 → 守土者
```

每卷字数约50万字，全书10卷共500万字。

## 实战经验总结

1. **知识库是防崩的核心** — 没有知识库，写到50章后专业术语就会重复和编造。有知识库，每章都能从不同条目中调用新干货
2. **progress.json是记忆系统** — 全程伏笔池、人物状态、知识库调用记录在一个JSON文件里，确保跨章节一致性
3. **70/30比例是甜区** — 纯剧情会干，纯干货会散。70%剧情拉动阅读欲望，30%干货提供获得感
4. **黄金三章决定生死** — 第一章300字内亮金手指，这个规则在平台没有例外
5. **每10章一个小高潮** — 读者需要持续的期待感，不能让节奏断掉
6. **伏笔回收是最强的爽点** — 当读者发现50章前埋的线索终于揭晓时，那种满足感比任何打脸都强
7. **人物对话风格必须锁定** — 全书里主角说的话必须像同一个人说的，不能前后判若两人
8. **全局节拍器** — 用一个数值（如频率/倒计时）作为全局节拍器，让读者有明确的"倒计时期待感"

## 快速启动指南

如果要从零启动一个新项目，按以下步骤操作：

1. **一句话创意** — 写下核心创意（如"退伍兵回乡继承风水祖业"）
2. **创意孵化** — 让孵化器生成5个版本方向，附概率评估
3. **爆款评分** — 对选定的方向进行100分制评分
4. **市场分析** — 调研目标平台的赛道、对标作品、读者偏好
5. **建知识库** — 上传相关PDF资料，提取文本，建立金句库
6. **写大纲** — 确定卷数、每卷主题、身份升级路线
7. **设人物** — 主角/女主/反派/配角的设定和对话风格锁定
8. **建progress.json** — 初始化进度追踪系统
9. **写黄金三章** — 第一章300字亮金手指，第三章必须引出核心悬念
10. **启动自动化** — 配置定时任务，每30分钟2章，持续产出
